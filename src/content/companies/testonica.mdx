The role of instruments is to verify PCBA hardware infrastructure: on-board interconnections and communication with peripheral components. In this way, every PCBA board can be checked for defects and stability issues. All instruments are pre-compiled for a target board and are being executed from test platform using predefined templates.

  <section className="space-y-4 text-slate-900">
    <h2 className="text-3xl font-bold">QI test firmware and application firmware synergy</h2>
    <div className="text-lg leading-relaxed">
      QI test firmware is used only <strong>temporarily</strong> during production / prototype / troubleshooting stage. It will populate the board when it is fresh (when it is just produced). QI test firmware runs on bare metal and will do just the testing and the diagnosis. It is not going to stay on a board when the board leaves an EMS factory. It will erase itself, so the user firmware (application firmware) will stay and will be shipped to the end customer. Normally, the QI test firmware loads the user's application firmware into the board as a last step on a production line.
    </div>

    <div className="text-lg leading-relaxed">

Simple I/O connectors, Ethernet links and high-speed interfaces, like LVDS, SMA, SFP or PCIe are some common examples. A matching board or as simple techniques as looping back some buses or adding some external testers (or matching boards) is the way to get complete coverage.

  <section className="space-y-4 text-slate-900">
    <h2 className="text-3xl font-bold">Key Features</h2>
    <ul className="list-disc space-y-2 pl-6 text-lg leading-relaxed">
      <li>Every test engineer and product manager knows how much time is often required to localize the problem. Sometimes it takes hours, sometimes days or even months to find the root cause. It is even worse if the product is already coated and packaged when you detect the failure.</li>
      <li>Using <strong>Quick Instruments</strong>, you can test the board component by component and gigabit links - line by line. In case of a failure, you know exactly which communication bus or line does not work properly. This considerably shortens the searching area and streamlines the repair. The important aspect is that this information is available <strong>before</strong> the board has left EMS production facilities and product owner does not receive defective PCBAs.</li>
      <li>All our instruments are pre-developed, fetched from the off-the-shelf instrumentation library and compiled for a particular board under test using our back-end compilation servers. As a result, ready-to-use test firmware is available in just half a day. You don't need an application firmware, no FPGA design tools and knowledge required.</li>
      <li>All our tests are optimized for short test duration and can be executed during the end-of-line manufacturing tests. The instruments are working in the same conditions which a board designer planned and in the same way as the FPGA designer will run the application. Standard test techniques like ICT, Boundary Scan, and Flying Probe in many cases are not capable to verify high-speed interfaces and components on a bare board with an FPGA. With <strong>Quick Instruments</strong>, tests are executed at full functional speed (at-speed testing).</li>
      <li>By its nature, Quick Instruments is a software product. Being embedded to the FPGA logic and by applying the test from the inside, Quick Instruments have a set of unique advantageous features: an access to <strong>internal</strong> on-board signals, no need for testpoints and nails, no signal integrity issues.</li>
      <li>Almost <strong>no price for a duplicate</strong>. You can scale the number of production lines and you only need a USB-to-JTAG cable.</li>
      <li>No need for maintenance or calibration, no degradation in performance (<strong>can't outdate</strong>).</li>
      <li><strong>Easier logistics</strong>: virtual Instruments that can be sent over email, copied and backed up.</li>
